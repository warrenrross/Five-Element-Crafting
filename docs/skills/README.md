# Skills

Long-form workflows for working on Five-Element-Crafting. Each file walks through one recurring task — the touchpoints, the procedure, and the gotchas.

For the index and the "where things live" map, start at [`SKILLS.md`](../../SKILLS.md) at the repo root.

## Files

1. [`01-local-dev-loop.md`](./01-local-dev-loop.md) — Serve, edit, hard-refresh, inspect state.
2. [`02-add-entity.md`](./02-add-entity.md) — Add a craftable noun to the catalog.
3. [`03-add-recipe.md`](./03-add-recipe.md) — Wire an actor/patient ordered pair to a result.
4. [`04-regenerate-data.md`](./04-regenerate-data.md) — Rebuild the JSON catalog from the design docs.
5. [`05-headless-smoke-test.md`](./05-headless-smoke-test.md) — Playwright-based scriptable verification.
6. [`06-mobile-and-touch.md`](./06-mobile-and-touch.md) — Exercise the touch-drag path and mobile layout.
7. [`07-add-ui-surface.md`](./07-add-ui-surface.md) — Add a panel, drawer, overlay, modal, or HUD element.
8. [`08-tune-balance.md`](./08-tune-balance.md) — Adjust deltas, difficulty bands, PCG biases.
9. [`09-deploy.md`](./09-deploy.md) — Publish to GitHub Pages.
10. [`10-agent-notes.md`](./10-agent-notes.md) — End-of-session handoff log.
11. [`11-add-mechanic.md`](./11-add-mechanic.md) — Introduce a new move type or resolution rule.

## When to add a new skill

If you find yourself doing the same kind of work for the third time and reaching for the same set of files and tools each time, that's a skill. Add it here.

Keep skills focused: one workflow per file, prescriptive procedure, real file paths and example code, an explicit "gotchas" section.
