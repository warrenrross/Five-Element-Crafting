# Five-Element-Crafting — v1 scaffold

A vanilla-JS, static-hosted prototype of the Wu-Xing crafting game described in `docs/design/`.

## Run it

This app uses ES modules and `fetch()` for its JSON data, so it must be served over HTTP — opening `index.html` with `file://` will not work.

From this directory:

```bash
python3 -m http.server 8000
```

Then open http://localhost:8000.

Any static server works (`npx serve`, `caddy file-server`, GitHub Pages, etc.). There is no build step.

## What works in v1

- Three-panel layout: discoveries on the left, free-placement workspace in the center, five phase tiles on the right (per `docs/design/ui-layout.md`).
- Drag a phase tile onto the workspace to spawn it.
- Drag one workspace entity onto another to craft. Ordered-pair lookup against `src/data/recipes.json` (110 recipes covering all 95 v1 entities).
- Move-type cues (`docs/design/ui-layout.md §3.3`):
  - Sheng → green halo + pulse-up ("Infuse")
  - Ke → red halo + press-down ("Consume")
  - Self → gold halo + brightness hold ("Concentrate")
  - Insub → purple halo + horizontal shake ("Disturb")
  - Stage-2 → neutral indigo + rotate
- Self-craft walks Feeling → Surge → Storm via a per-element counter (`engine/self-state.js`).
- Null drops snap back and show a label for the first three nulls in a session.
- Catastrophe drops trigger a ~2-second lockout overlay (Explore mode).
- Discoveries ledger persists in `localStorage` under `fec.discoveries.v1`.
- Clicking a discovered entry in the left panel spawns a copy on the workspace and opens inspect. Shift-click inspects without spawning.
- Inspect drawer shows Surface lore, phase-weight chips, and "Made by" recipes.
- Clear button with confirm modal.

## Not in v1 (intentional)

- Balance mode and the pentagram readout — see `docs/design/balance-mode.md` for the spec; the workspace already includes a hidden `#pentagram-background` element to extend into.
- Reading-lore (long-form citation-rich text) — lives in `docs/design/inspect-copy.md`; not loaded into the JSON catalog yet.
- Audio/haptics.

## Layout

```
app/
  index.html
  src/
    main.js                  # entry: wires everything
    engine/
      recipes.js             # data loader + resolve() + discoveries ledger
      self-state.js          # per-element self-stage counter
    ui/
      phase-panel.js         # right panel
      workspace.js           # center, drag/drop, craft resolve, cues
      discoveries.js         # left panel
      inspect.js             # right-edge inspect drawer
    data/
      entities.json          # 95 entities (id, name, emoji, tier, weights, lore)
      recipes.json           # 110 ordered-pair recipes
      self_progression.json  # Feeling → Surge → Storm by element
      catastrophes.json      # catastrophe ids for lockout
    styles/
      main.css               # all styles
```

## Regenerating the data files

The JSON data files were generated by `/tmp/build_data.py` (not committed). If you change the entity table in `docs/design/stage-2-crafts.md`, re-run that script to regenerate them.
